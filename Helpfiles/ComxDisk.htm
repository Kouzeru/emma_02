Disk Support

To use disk images select fdc.bin in any available slot (CARD 1 if no expansion box is used). At least one COMX disk image is needed (as DISK 1) to boot COMX DOS. The Emulator has installed the master disk image 'dos.1.4+f&m.disk.tools.img'. When DOS NEW is used the COMX DOS should be initiated with the message: 

COMX DOS  V1.4
COPYRIGHT BY COMX - 1984

After this any DOS command or utility from the master disk can be used, for explanation of COMX DOS commands please see the COMX DOS manual available on the Dutch COMX Club.

To change the disks at run time just select a different filename via the DISK x button or type a name in the text field. To eject the disk use the eject button or remove the filename in the text field.

Disk Format and boot initialization

To create a new disk image including COMX DOS boot data, keep 'dos.1.4+f&m.disk.tools.img' specified as DISK 1 and any filename as DISK 2. After using DOS URUN,"INIT" the format/init program will be started, press any key and 'Y' for being sure, then select drive no: 2. After this 3 Different type of disks can be created by selecting the options:

1. Double sided disk, 35 tracks per side with a total size of 140K:

- Double track density: N
- Double sided: 2

2. Single sided disk, 70 tracks with a total size of 140K:

- Double track density: Y

3. Single sided disk, 35 tracks with a total size of 70K:

- Double track density: N
- Double sided: 1

After this selection, again confirm that you are sure and the format process will start.

F&M Disk Utilities and utilities not documented in COMX DOS Manual

DISKCOPY

Make an exact copy of a disk, including a format of the destination disk before copy begins.

FILECOPY

Copy any file to another disk image

F&M COPYDUB

F&M Disk copy program copying a complete disk; however the destination disk will have to be formatted in advance via the INIT program as described above.

F&M LOCK

Locks all software on the disk.

F&M ULOCK

Unocks all software on the disk.

F&M DISKMONITOR

Allows editing and inspecting disk data directly on the disk or in the emulator case directly in the disk image.

F&M INIT

Formats a disk like the INIT program however with an F&M boot sector which will attempt to load and run a program called 'STARTUP' when DOS is initialized (like with DOS NEW).

STARTUP

An example start-up program showing all programs on disk and allowing starting them from a menu.
