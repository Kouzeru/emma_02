Elf EMS Memory and RAM Pager

To switch Elf EMS memory page from the debugger use the 'EMS' field (hexadecimal values 0 to 1F). EMS pages are connected from address hex 8000-BFFF.

To switch Elf memory pager from the debugger use the 'Pager' fields (first field values 0 to F, second field values 0 to FF). Memory pager pages are connected from address hex 1000 to hex FFFF in blocks of 4K. Page 0 is always connected from address 0 to hex FFF. The switchable 4K page is indicated by the fist field and the selected pager by the second field.

The switchable EMS address will be shown in the same colour as the 'EMS' text and the switchable Pager pages in the same colour as the 'Pager' text. When switching the switchable 4K page the colour will move as well.

Save

To save memory values within the CPU or video memory space, use the 'Save' button. The full memory will be saved, i.e. 64K in CPU mode, for video modes size is depending on setting of video RAM size.

To save in binary format use file extension '.bin' and to save in text format use file extension '.txt'. If the specified file already exists the filename will be given a number starting with 1.

Copy

To copy memory values within the CPU or video memory space use the 'Copy' button. First make sure to specify start and end address of the source and start (to) address of the destination location, all in hexadecimal format.
