Keyboard Mapping and Types

This chapter describes Elf keyboard mapping for details, for VT100 terminal key mappings see chapter VT100 Setup & Keymapping

Keyboard Mapping

Default the Elf 'In' button is mapped to the PC Insert key on Windows and PC Control key on Mac OSX. F12 is mapped to the Elf 'Run' switch, so if the CPU is stopped it will start with F12 and when running it will stop with F12, stop mode is indicated in the emulated window title. For the super Elf, F12 simulates the 'R' and 'G' button. See also Keypad Mapping and Function and Hot Keys.

Keyboard Types

To select the specific Elf keyboard type use the choice button 'Keyboard'. Currently 5 types are supported:

PS2 Keyboard:

Keyboard type operating with serial communication using control characters

PS2 Interrupt:

Same as PS2 Keyboard but additionally triggering an interrupt at every key press

Ascii:

Ascii Keyboard directly connected to an 1802 input port.

Ascii (+Hex):

Ascii Keyboard directly connected to an 1802 input port. When this option is selected also an Hex keypad with EF (default EF3) connection is emulated at the same time.

Hex (+EF) Keyboard:

Only 1 hex keypad is supported. Both the Netronics Elf II and the Quest Super Elf already have hex keyboard support on the main board however this will not set an EF (default EF3) to 0 when a key is pressed. When selecting the Hex (+EF) keyboard option an EF will be set to 0 when a key is pressed. This allows for running some games like 'space invaders'.

To change and view the hex keypad to PC key mapping select the 'Key Map' button, see also chapter Key Mapping

Hex Keyboard:

This option is only valid for the Cosmac Elf and is the same as described above however without an EF being set to 0 when a key is pressed.
